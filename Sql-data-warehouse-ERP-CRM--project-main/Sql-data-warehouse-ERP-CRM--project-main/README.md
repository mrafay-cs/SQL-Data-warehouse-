# Sql-data-warehouse-ERP-CRM--project
End-to-end modern data warehouse development in SQL Server using Medallion Architecture, ETL, data modelling, and analytical reporting.
Welcome to the **Data Warehouse and Analytics Project** repository
This project demonstrates the design and implementation of a modern data warehouse solution in **SQL Server** using the Medallion Architecture approach. The warehouse is structured into Bronze, Silver, and Gold layers to organize raw, cleansed, and curated data for scalable analytics and reporting.

The solution includes ETL processes to extract data from source systems **ERP & CRM**, perform data cleansing and transformations, and load it into structured warehouse layers. It also applies data modeling techniques to create analytics-ready datasets that support business intelligence, reporting, and dashboarding.

This project showcases key data engineering and BI concepts such as data ingestion, transformation, dimensional modeling, data quality handling, and analytical data preparation, making it a strong end-to-end example of building a SQL Server-based data warehouse for modern reporting needs.

**Project Specifications**

This project is divided into two key phases:
Building the Data Warehouse (Data Engineering)
BI: Analytics & Reporting (Data Analysis)
It is designed to simulate a real-world data project by covering both the data engineering layer and the business intelligence / analytics layer.
## Building the Data Warehouse (Data Engineering)
**Objective :**
Developed a modern data warehouse in SQL Server to consolidate sales data from multiple source systems like ERP & CRM and provide a reliable foundation for analytical reporting and data-driven decision-making.

**Specifications**

**Data Sources:**
Import data from two source systems (ERP and CRM) provided as CSV files.

**Data Quality:**
Perform data cleansing and resolve data quality issues before loading data into analytical layers.

**Data Integration:**
Combine and transform data from both source systems into a single, user-friendly analytical data model optimized for querying and reporting.

**Scope:**
Focus only on the latest available dataset. Historical tracking / data historization is out of scope for this project.

**Documentation:**
Provide clear and structured documentation of the data model, transformations, and warehouse layers to support both business stakeholders and analytics teams.

## BI: Analytics & Reporting (Data Analysis) 
**Objective:**
Develop SQL-based analytical solutions to generate meaningful business insights and support strategic decision-making.

**Analytics Focus Areas**
The reporting and analysis layer is designed to provide detailed insights into:
Customer Behavior - 
analyze customer activity, purchasing patterns, segmentation, and contribution to revenue.
Product Performance -
Evaluate product-level sales, category performance, top-performing products, and product trends.
Sales Trends
Examine sales growth, order patterns, seasonal trends, and key performance indicators over time.

## Business Value 
These insights enable stakeholders to monitor key business metrics, identify opportunities for improvement, and make data-driven strategic decisions.
